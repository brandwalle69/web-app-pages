# web-app-pages

本仓库用于归档和发布多个独立 HTML 页面。这些页面通常用于展示、测试或作为轻量级 Web 应用的静态入口，不依赖于特定域名或后端服务。

## 项目简介

web-app-pages 是一个集中存放 HTML 文件的仓库，每个文件代表一个独立的网页或应用界面。仓库本身不限定页面的具体用途，可以是工具页面、演示页面、原型页面或其他任何适合静态部署的 HTML 内容。

通过 GitHub Pages 或其他静态托管服务，这些页面可直接通过 URL 访问，适合快速分享或作为前端片段存档。

## 目录说明

```
web-app-pages/
├── pages/          # 存放所有独立 HTML 页面文件
├── assets/         # 公共资源文件（CSS、JS、图片等）
└── README.md       # 本文件
```

- **pages/**：每个 `.html` 文件对应一个独立页面。文件名应体现页面功能或内容，避免歧义。
- **assets/**：存放页面间共享的静态资源，如图片、样式表或脚本。若页面资源独立，建议直接放在页面所在目录或内联在 HTML 中。

## 页面归档说明

- 每个页面应自包含，尽量不依赖外部资源（除公共库外），以保证长期可访问。
- 页面之间相互独立，不共享状态或逻辑。
- 若页面需要外部依赖（如 CDN 库），请在 HTML 头部明确标注来源和版本。
- 归档时建议在页面内添加简短注释，说明创建时间、用途或作者。

## 维护说明

- 新增页面：在 `pages/` 下创建新的 `.html` 文件，遵循命名规范（如 `example-page.html`）。
- 更新页面：直接修改对应文件，并更新文件内注释（如有）。
- 删除页面：移除文件并确认无其他页面引用。
- 公共资源变更：谨慎修改 `assets/` 下的文件，避免影响多个页面。

欢迎提交 Pull Request 添加新的页面或改进现有内容。如有疑问，请通过 Issues 联系。
